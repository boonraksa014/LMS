# Product

## Register

product

## Users

พนักงานในองค์กร (corporate employees) ใช้ระบบเพื่อฝึกอบรมและพัฒนาทักษะในเวลางาน มี 3 บทบาท:

- **ผู้เรียน** — ต้องการดูความคืบหน้า, เรียนบทเรียน, ทำแบบทดสอบ, รับใบรับรอง โดยไม่เสียเวลากับ UI ที่ซับซ้อน
- **ผู้จัดการ** — ติดตาม progress ของทีม, ดูรายงาน, จัดการกลุ่มผู้เรียน
- **แอดมิน** — สร้างและจัดการคอร์ส, ผู้ใช้, เทมเพลตใบรับรอง

ใช้งานในบริบท corporate — ระหว่างเวลางาน, บน desktop เป็นหลัก, ต้องการความรวดเร็วและความชัดเจน

## Product Purpose

ระบบจัดการการเรียนรู้ (LMS) ภาษาไทยสำหรับองค์กร — ให้พนักงานเรียนรู้ผ่านคอร์สออนไลน์ ทำแบบทดสอบ และรับใบรับรอง ส่วนผู้บริหารติดตามผลและจัดการหลักสูตรได้ในที่เดียว

ความสำเร็จ = พนักงานเรียนจบคอร์สและสอบผ่านได้โดยไม่มีอุปสรรคจาก UI; ผู้จัดการเห็น progress ของทีมได้ชัดเจน

## Brand Personality

มืออาชีพ, ทันสมัย, ชัดเจน

เสียง: ตรงไปตรงมา ไม่เป็นทางการเกินไป, สนับสนุนผู้เรียน, ไม่เล่นๆ
ความรู้สึกที่ต้องการ: ไว้วางใจได้ + กระตุ้นให้อยากเรียนต่อ

## Anti-references

- **Generic SaaS cream/purple templates** — สีผิวเลา gradient ม่วงแบบ 2023 SaaS ที่เหมือนกันหมด
- **Childish / cartoon tone** — ไม่เหมาะกับบริบท corporate
- **Moodle-style legacy LMS** — dense layout, boring, ดูล้าสมัย, ไม่ motivating
- **Overloaded UI** — ข้อมูลเยอะเกินไป ทำให้ไม่รู้ว่าต้องทำอะไร

## Design Principles

1. **Progress is always visible** — ผู้เรียนต้องเห็นได้ทันทีว่าตัวเองอยู่ที่ไหนในการเรียน เป็นแรงจูงใจหลัก
2. **Trusted structure** — hierarchy และ layout ต้องคาดเดาได้ ผู้ใช้ไม่ควรต้องเรียนรู้ UI ใหม่
3. **Corporate warmth** — จริงจังพอสำหรับองค์กร, อบอุ่นพอให้ไม่รู้สึกเหนื่อยกับการเรียน
4. **Focus over density** — แสดงสิ่งที่สำคัญที่สุดก่อน ซ่อนสิ่งรองลงไป
5. **Thai-first** — ออกแบบให้ภาษาไทยและ Sarabun อ่านง่ายสบายตา ไม่ใช่ adapter ของ design ภาษาอังกฤษ

## Accessibility & Inclusion

- WCAG AA — contrast ratio ≥ 4.5:1 สำหรับ body text, ≥ 3:1 สำหรับ large text
- รองรับ prefers-reduced-motion
- Keyboard navigation สำหรับ core workflows
- ภาษาไทย (Sarabun) ต้องมี line-height เพียงพอ (ตัวอักษรไทยต้องการ space มากกว่า Latin)
